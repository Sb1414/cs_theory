## Что такое порт ?
***Порт*** — целое неотрицательное число, указываемое при запросе на соединение, записываемое в заголовках протоколов транспортного уровня сетевой модели OSI (TCP, UDP, SCTP, DCCP).

Для приема и передачи данных.


## Что такое HTTP ?
HyperText Transfer Protocol — «протокол передачи гипертекста» в формате HTML.

сетевой протокол передачи данных. Служит для отправки запросов веб-серверу и получения ответов в виде [HTML](#что-такое-html) или файлов. По умолчанию, HTTP работает на сетевом **порту 80**.

Является небезопасным протоколом, так как все данные передаются в открытом виде. Как альтернатива, используется похожий HTTPS, который шифрует информацию.

**Заголовки HTTP-протокола**

- Основные заголовки (General) содержит такую информацию как URL, METHOD, STATUS-CODE
- Заголовки запроса (Request Headers) содержит больше информации о клиенте, такую как куки, браузер, язык.
- Заголовки ответа (Response Headers) содержат информацию об ответе, например его местонахождение, или о сервере, предоставившем его.
- Тело запроса (Body) содержат информацию о теле ресурса(JSON)


## Что такое HTTPS ?
расширение протокола HTTP с поддержкой шифрования. Используется для безопасной передачи конфиденциальных данных в сети. По умолчанию, HTTPS работает на сетевом **порту 443**.

Пример использования — страницы веб-сайтов с необходимостью ввода логина и пароля


## Что такое HTML ?
язык разметки для создания и придания структуры веб-страницам. Представлен тегами — служебными наборами символов для указания параметров элемента.

## Что такое FTP ?
протокол для реализации передачи файлов по сети. Для использования необходимо специальное программное обеспечение, называемое FTP-клиентом (например, браузер или Filezilla), которое подключается к FTP-серверу. По умолчанию, используются сетевые порты 21, 20 (активный режим), 49152-65534 (динамические).

Может работать в двух режимах — активном и пассивном. При активном клиент отправляет свой IP-адрес, сервер создает с ним соединение. Пассивный — клиент получает от сервера номер динамического порта и устанавливает с ним соединение по этому порту. Пассивный режим применяется клиентами, находящимися за брандмауэром.

## Что такое SMTP ?
протокол для обмена электронной почтой. Расшифровывается как Simple Mail Transfer Protocol. По умолчанию, работает на портах 25, 465 (требует SSL) или 587 (STARTTLS, требует аутентификацию).

SMTP, в отличие от других почтовых протоколов, таких как [POP3](#что-такое-pop3) или [IMAP](#что-такое-imap) не участвует в доставке почтовых сообщений в почтовую программу пользователя. Он используется только для отправки сообщений между серверами или сервером и почтовым клиентом.

Чтобы подключиться к серверу SMTP и выполнить отправку сообщения, не обязательно наличие специальных программ. Можно поступить проще — подключиться к серверу, используя telnet. Пример команды:
```
telnet smtp.mail.ru 25
```
* где 25 — порт, на котором слушает сервер, к которому мы решили подключиться.

## Что такое POP3 ?
протокол для получения электронной почты с почтового сервера. Расшифровывается как Post Office Protocol Version 3. По умолчанию, работает на сетевом TCP/IP порту 110 (без шифрования) или по 995 (с TLS или SSL шифрованием).

## Что такое IMAP ?
протокол для управления ящиком электронной почты. Расшифровывается как Internet Message Access Protocol (протокол доступа к сообщениям в Интернете). По умолчанию, может работать на 2-х TCP портах: 143 (без шифрования или STARTTLS) и 993 (over SSL).

## Что такое javascript ?
Язык программирования. Большое применение нашел при разработке динамических веб-сайтов с интерактивностью на стороне клиента (браузера). Настолько широко распространен, что если отключить поддержку этой технологии, многие сайты перестанут нормально работать.

Простой пример строчки кода Javascript — alert('Hello World'); — вызовет всплывающее предупреждение с текстом "Hello World";

Javascript встроен в браузер и не требует скачивания, установки и обновления. 


## Чем отличается метод POST от метода PUT ?
- Метод POST всегда создаёт новую запись в БД. 
- Метод PUT проверяет есть ли запись в БД, если есть, то он её редактирует (обновляет) её, если нет, то создаёт новую запись.

## Чем отличается метод GET от метода POST ?
- Данные методом GET передаются в строке запроса (т.е. в URL-адресе). 
- Данные методом POST передаются в теле (body) запроса.

## Какие группы статус кодов вы знаете ?
- Информационные 100е (в обработке)
- Успешные 200е (успешно обработан)
- Перенаправления 300е (перенаправлено и найдено)
- Клиентские ошибки 400е (не найдено)
- Серверные ошибки 500е (внутренняя ошибка сервера)


Самые популярные ответы сервера (коды состояния и статусы):
- 200 OK в случае успешной обработки запроса.
- 301 Moved Permanently если редирект используется на постоянной основе.
- 307 Temporary Redirect если редирект используется временно.
- 400 Bad Request если в запросе есть синтаксическая ошибка.
- 403 Forbidden если запрос успешный, но сервер его не может выполнить, поскольку пользователь не имеет достаточных прав.
- 404 Not Found если запрошенного ресурса не существует.
- 500 Internal Server Error если работа программы на сервере выдала ошибку.

## Про JSON
JSON (JavaScript Object Notation) — текстовый формат обмена данными, основанный на JavaScript. Но при этом формат независим от JS и может использоваться в любом языке программирования. Это коллекция пар типа ключ-значение. 

## Про REST
REST – архитектурный стиль реализации клиент-серверных приложений на основе протокола HTTP.

REST (Representational State Transfer) - это архитектурный стиль разработки программного обеспечения, который определяет набор ограничений и принципов для построения распределенных систем. RESTful веб-API обеспечивает взаимодействие между клиентом и сервером с использованием стандартных HTTP-методов, URL-адресов и представлений ресурсов.

Основные принципы REST включают:

- Ресурсы: Ключевая абстракция в REST - это ресурс, представляющий собой какую-то сущность, к которой можно обратиться по URI (Uniform Resource Identifier).

- Представление: Ресурсы могут иметь различные представления, такие как JSON, XML или HTML. Клиенты взаимодействуют с ресурсами, обмениваясь их представлениями.

- Состояние: Клиент и сервер могут обмениваться представлениями ресурсов, и состояние клиента хранится на клиенте, не на сервере. Каждый запрос клиента должен содержать всю необходимую информацию для понимания запроса сервером (без сохранения состояния на сервере между запросами).

- Единообразие интерфейса: Интерфейс взаимодействия между клиентом и сервером должен быть единообразным. Это включает в себя ограниченный и заранее определенный набор операций (например, GET, POST, PUT, DELETE), которые применяются к ресурсам.

- Без состояния: Каждый запрос клиента к серверу должен содержать всю необходимую информацию для понимания и обработки запроса. Сервер не должен хранить состояние клиента между запросами.

REST широко используется в веб-разработке для построения веб-сервисов, в том числе API (интерфейсов программирования приложений), которые позволяют взаимодействовать с серверами и обмениваться данными между клиентскими приложениями и серверами.

## Как тестировать API ?
- Postman - это HTTP-клиент для тестирования API. HTTP-клиенты тестируют отправку запросов с клиента на сервер и получение ответа от сервера.
- Swagger - язык описания интерфейсов для описания RESTful API, выраженных с помощью JSON. Программа которая сочетает в себе документацию и из него же можно слать запросы.

## Какие вкладки в инструментах разработчика вы знаете ?
- Elements – структура и стиль страницы (HTML и CSS), смотрим размеры элементов, отступы, шрифты, цвет
- Console – здесь отображается информация об ошибках в коде. Если на странице не подгрузились шрифты, картинки или стили, то сообщения об ошибках выводятся в консоль.
- Application - сброс куки и кэша
- Network – для того чтобы посмотреть, как отработал тот или иной запрос. 

```
Как пользоваться Network:
Находили нужный запрос, смотрим его URL, метод, тело запроса, сверяем с документацией, чтобы убедиться, что он правильно сформирован. Далее смотрим на статус код и ответ сервера. С помощью статус кода мы можем увидеть результат операции, а в ответе сервера смотрим полученную с сервера информацию.
```
- Кнопка переключения устройств – для того чтобы посмотреть как будет отображаться наше приложение на различных разрешениях экрана.

## Уровни оси
- физический: кабели, кодирование единиц и нулей. Это уровень проводов. Эзернет, который описывает как эти сигналы должны кодироваться, передаваться по проводам и тд. Еще на этом уровне, блютуз, вф, инфракрасный порт. Сетевые устройства первого уровня - это концентраторы и репиторы, т.е. глупые железки, которые могут работать с физич. сигналом, не вникая в его логику, а просто передавая пришедший сигнал
- канальный: мы получили сигнал с 1 уровня (это набор напряжений разной амплитуды, волн или радиосигнала), на 2 уровне нужно это закодировать в биты, которые комп сможет понять. На 2 уровне проверяются и исправляются ошибки передачи, есть фрейм, у него данные об отправителе и получателе, это mac адреса, они состоят из 48 бит и выглядят примерно так: 00:16:52:00:1F:03. Он нужен чтобы идентифицировать устройства в одной сети. Т.е. МАК - это адрес устройства на канальном уровне. На этом уровне обитают такие устройства как коммутаторы и мосты. Нужны для передачи фрейма нужному адресату. Стандарт эзернет тоже есть. Протоколы PPP CDP MPLS 

- сетевой уровень (маршрутизация, ip адрес). На этом уровне маршрутизация трафика. Если мы ходим попасть в бузу wiki.ru , то мы отправляем DNS запрос , получаем ответ в виде ip адреса и подставляем его в пакет. (если на 2 уровне мы использует термин ФРЕЙМ, то здес уже ПАКЕТ). Письме уже написаны написаны адреса другого уровня, сетевого: 168.212.226.204. Именно тут при момощи ICMP используем утилиту PING, который помогает определить наличие проблем сети.  Из устройств здесь живет маршрутизатор и его протоколы маршрутизации: BGP, OSPF, RIP, EIGRP. Процесс, когда данные передаются в верхнего уровня на нижний - называется инкапсуляцией данных. С нижних на верхний - декапсуляция.

- транспортный уровень - обеспечивает передачу данных по сети. Здесь TCP и UDP. Различный транспорт для разной категории трафика. Трафик чувствителен к потерям - ***TCP*** (transmission control protocol). Обеспечивает контороль за передачей данных. Передает без ошибок и в том виде, в котором и был изначально. Немного потерь - ***UDP*** (user datagram protocol). Например при просмотре видео пара пакетов могло потеряться - не критично.

- сеансовый уровень управляет соединениями. Например аудио-видео конфа, тк на этом уровне устанавливается  каким кодеком будет кодироваться сигнал. Причем этот кодек должен присутствовать у обоих пользователей

- уровень представления - преобразование форматов сообщений / кодирование, сжатие (JPEG, GIF)

- уровень приложения - сетевые службы позволяют пользователям серфить просторы интера: это https (http, ftp, smtp). Уровень нужен чтобы приложения имели доступ к сетевым службам, таким как доступ к файлам, пересылки писем, запросы к бд

## Вы набираете google.com в браузере . Расскажите как можно подробнее, что происходит в это время на HTTP-уровне ?
1. DNS-разрешение:
    - Браузер отправляет запрос на DNS-сервер для получения IP-адреса, связанного с доменным именем "google.com".
    - DNS-сервер возвращает IP-адрес (например, 216.58.204.46).
2. Установление TCP-соединения:
    - Браузер устанавливает TCP-соединение с сервером, используя полученный IP-адрес и порт 80 (стандартный порт для HTTP) или 443 (если используется HTTPS).
3. Отправка HTTP-запроса:
    - Браузер отправляет HTTP-запрос к серверу Google, содержащий метод запроса (обычно "GET"), запрашиваемый путь ("/") и другие заголовки запроса.
4. Серверная обработка:
    - Сервер Google принимает HTTP-запрос и обрабатывает его. В случае с "google.com" сервер может выполнить поиск или вернуть главную страницу.
5. Отправка HTTP-ответа:
    - Сервер формирует HTTP-ответ, содержащий статус ответа (например, 200 OK), заголовки ответа и сам контент страницы.
6. Передача данных:
    - Сервер отправляет HTTP-ответ обратно по установленному TCP-соединению.
7. Отображение контента:
    - Браузер получает HTTP-ответ и начинает обрабатывать контент. Если ответ содержит HTML, браузер начинает разбирать и отображать веб-страницу.
8. Дополнительные запросы:
    - В ходе обработки HTML-страницы браузер может обнаруживать дополнительные ресурсы (например, изображения, стили, скрипты) и отправлять дополнительные HTTP-запросы для их получения.
9. Завершение соединения:
    - После завершения передачи данных браузер и сервер могут решить разорвать TCP-соединение (в случае HTTP/1.0) или сохранить его открытым для возможных дополнительных запросов (в случае HTTP/1.1 с поддержкой keep-alive).
